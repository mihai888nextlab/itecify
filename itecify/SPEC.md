# iTECify - Code Collaboration & Sandboxing Platform

## 1. Concept & Vision

iTECify este un workspace de code-collaboration unde oamenii și agenții AI lucrează simultan în același fișier, fără conflicte Git. Platforma combină eleganța editorului Notion cu puterea sandbox-urilor Docker izolate. Experiența trebuie să simtă că codezi în viitor - fluid, intuitiv, și collaborativ în timp real.

## 2. Design Language

### Aesthetic Direction
Dark cyberpunk minimal - un mix între terminalul clasic și interfețele sci-fi moderne. Accent pe claritate și contrast între codul uman (subtil, natural) și cel AI (vizibil, bloc-based).

### Motion Philosophy
- Cursor movements: spring animation (0.3s, ease-out)
- Block transitions: slide + fade (0.2s)
- Panel slides: cubic-bezier(0.4, 0, 0.2, 1)
- Loading states: pulse animation with gradient shimmer

### Visual Assets
- Icons: Lucide React (consistent stroke width)
- Code syntax: One Dark Pro theme with custom AI block highlighting
- Cursors: Custom SVG with name labels and color coding
- AI blocks: Gradient border with glow effect

## 4. Features & Interactions

### Core Features

#### 4.1 Multi-Cursor Collaboration
- Real-time cursor positions via Yjs + WebSocket
- Each user gets unique color and cursor label
- Smooth cursor interpolation (spring animation)
- Cursor labels show username and current selection
- Cursors fade after 3s of inactivity

#### 4.2 AI Block Editor
- AI-generated code wrapped in collapsible blocks
- Visual treatment: purple gradient border, subtle glow
- Block header shows: "Generated by [Agent Name]" + timestamp
- Controls on hover: Accept (green), Reject (red), Move (blue)
- Accepting: block becomes normal code with subtle fade animation
- Rejecting: block collapses with red flash, code removed
- Blocks can be nested (AI can generate inside AI blocks)

#### 4.3 Docker Sandboxing
- On-demand container creation via Dockerode
- Supported languages: Node.js, Python, Rust, Go, Java
- Image pre-caching for common runtimes
- Container lifecycle: Create → Scan → Run → Stream → Stop
- Resource limits: 512MB RAM, 1 CPU, 30s timeout

#### 4.4 Security Scanning
- Pre-run scan using pattern matching for:
  - eval() / exec() calls
  - os.system(), subprocess calls
  - File system access patterns
  - Network requests (suspicious)
  - Import of dangerous modules (os, sys, ctypes)
- Quarantine mode: run in read-only, no-network container
- User notification with severity levels (low/medium/high)

#### 4.5 Real-time Output
- WebSocket streaming of stdout/stderr
- ANSI color support in terminal
- Auto-scroll with manual override
- Clear output button
- Copy output to clipboard
- Filter by info/warn/error

### Side Quests

#### 4.6 Collaborative Terminal
- Shared xterm.js instance
- Command history visible to all participants
- Color-coded commands by user
- Input disabled when someone else is typing

#### 4.7 Time-Travel Debugging
- Yjs snapshots saved every 30s
- Timeline scrubber showing document evolution
- Click to preview any point in history
- Restore from snapshot option
- Diff view between snapshots

#### 4.8 Smart Resource Limits
- Memory limit: 512MB (configurable)
- CPU limit: 1 core (configurable)
- Execution timeout: 30s default
- Infinite loop detection via instruction count
- Graceful termination with user notification

## 6. Technical Approach

### Frontend Stack
- **Framework**: Next.js 16 (Pages Router)
- **Styling**: Tailwind CSS 4
- **Editor**: CodeMirror 6 with custom extensions
- **Terminal**: Xterm.js + xterm-addon-fit
- **Real-time Sync**: Yjs + y-websocket
- **State Management**: Zustand (lightweight)
- **Icons**: Lucide React

### Backend Stack
- **Runtime**: Node.js with TypeScript
- **WebSocket Server**: y-websocket server
- **Docker Integration**: Dockerode SDK
- **Database**: PostgreSQL (via Prisma) for metadata
- **Yjs Persistence**: Binary snapshots in PostgreSQL

### API Design

#### REST Endpoints
```
POST /api/sessions          - Create new session
GET  /api/sessions/:id      - Get session details
PUT  /api/sessions/:id       - Update session metadata
DELETE /api/sessions/:id     - Delete session

POST /api/containers         - Create container for session
POST /api/containers/:id/run - Execute code in container
POST /api/containers/:id/stop - Stop container
GET  /api/containers/:id/logs - Get container logs

POST /api/snapshots          - Save snapshot
GET  /api/snapshots/:id      - Get snapshot content
GET  /api/snapshots          - List snapshots for session
```

#### WebSocket Messages
```
// Client → Server
join_session: { sessionId, user }
leave_session: { sessionId }
cursor_move: { position, selection }
block_action: { blockId, action: 'accept'|'reject'|'move' }
execute_code: { language, code }

// Server → Client
user_joined: { user }
user_left: { userId }
cursor_update: { userId, position, selection }
block_created: { block }
block_updated: { block }
container_output: { type: 'stdout'|'stderr', data }
container_status: { status, message }
```

### Data Model

```typescript
// Session
interface Session {
  id: string;
  name: string;
  createdAt: Date;
  ownerId: string;
  participants: User[];
  settings: SessionSettings;
}

// User
interface User {
  id: string;
  name: string;
  color: string;
  role: 'human' | 'ai';
  cursorPosition?: Position;
}

// AIBlock
interface AIBlock {
  id: string;
  sessionId: string;
  agentId: string;
  content: string;
  status: 'pending' | 'accepted' | 'rejected';
  createdAt: Date;
  startLine: number;
  endLine: number;
}

// Container
interface Container {
  id: string;
  sessionId: string;
  language: string;
  status: 'created' | 'running' | 'stopped' | 'error';
  resourceLimits: {
    memory: string;
    cpu: number;
    timeout: number;
  };
}

// Snapshot
interface Snapshot {
  id: string;
  sessionId: string;
  content: Uint8Array; // Yjs binary
  timestamp: Date;
  description?: string;
}
```

### Security Considerations
- All WebSocket connections authenticated
- Code execution in isolated containers with no network
- Rate limiting on API endpoints
- Input sanitization for all user content
- Docker socket protected by Unix permissions
